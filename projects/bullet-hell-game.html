Bullet Hell Game

C# MonoGame Game Dev
View on GitHub

Overview

This Greek mythology-themed bullet hell game is a 2D shooter developed as part of the Software Design (CPTS 487) class at Washington State University. Built using the MonoGame/XNA framework, it challenges players to dodge waves of enemy projectiles while progressing through different mythological encounters.

Problem Statement

Creating a bullet hell game involves solving complex technical challenges including efficient collision detection with many entities, managing game state, implementing enemy AI with diverse attack patterns, and ensuring smooth performance even with hundreds of projectiles on screen simultaneously.

Key Features

Dynamic Wave System

The game features a JSON-driven wave system that controls enemy spawning and behavior patterns. This data-driven approach allows for easy creation of new levels and enemy types without modifying code. Each wave progressively increases in difficulty, challenging players to improve their skills.

Boss Encounters

The game includes a mid-boss (Minotaur) and final boss (Zeus) with unique attack combinations and patterns defined in separate configuration files. These challenging encounters test player reflexes and pattern recognition with complex bullet patterns and phase-based behaviors.

Power-up System

Players can collect various power-ups dropped by defeated enemies to enhance their capabilities. These include health restoration, temporary shields, and damage boosts. The power-up system adds strategic depth as players must decide when to risk collecting items during intense battles.

Design Principles

The game is built around several core design principles:

Thematic Cohesion Ancient Greek mythology provides the aesthetic foundation with enemies, environments, and bosses all derived from Greek legends.
Progressive Difficulty Challenges increase gradually with carefully designed wave sequences building toward boss encounters.
Varied Enemy Types From Harpies and Satyrs to Cyclopes and Gorgons, each enemy features unique movement and attack patterns.
Bullet Pattern Diversity Multiple bullet types (basic, spread, homing, burst, and beam) provide varied gameplay challenges.
Visual Clarity Art style inspired by ancient Greek pottery designs, ensuring player can distinguish between threats despite intense action.

Models Game entities and core logic (Player, Enemy, Bullet, etc.)
Views Render game objects and effects to the screen

Implementation Challenges

Collision Detection Optimization

With potentially hundreds of bullets on screen simultaneously, naive collision detection would cause significant performance issues. We implemented optimized collision detection systems to maintain performance while accurately detecting hits between players, enemies, and projectiles.

Complex Enemy Behavior Patterns

To create engaging enemy behavior, we designed a pattern system that combines basic movements with various attack patterns. These components are configured through JSON files to create diverse and challenging enemies without code changes.

Game Controls

WASD / Arrow Keys Move the player
Spacebar Shoot
E key Slow movement toggle (for precision dodging)
Escape Pause game

Outcome & Learnings

This project provided valuable experience in game development principles, including game loops, sprite rendering, collision detection, and state management. The data-driven approach to enemy waves and patterns created a flexible system that could be expanded with new content without significant code changes.

Performance optimization became a key learning area, as maintaining good framerate with many entities required careful attention to memory allocation patterns and efficient algorithms. The team-based development process also provided valuable experience in collaborative software design and implementation.
