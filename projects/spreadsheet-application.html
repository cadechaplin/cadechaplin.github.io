Overview

This Spreadsheet Application is a cross-platform desktop application built with C# and Avalonia UI that provides core spreadsheet functionality with formula support, cell referencing, and error handling. It features a clean, intuitive interface with undo/redo capabilities and multi-cell selection.

Problem Statement

Many spreadsheet applications are either platform-specific or overly complex. The goal was to create a lightweight, cross-platform solution that focuses on core spreadsheet functionality while maintaining a clean code structure and proper error handling.

Key Features

Formula Support & Cell Referencing

The application supports mathematical formulas and references to other cells. Users can create complex calculations that automatically update when referenced cells change. The formula parsing system handles parentheses, operators, and functions with proper precedence rules.

Error Handling & Circular Reference Detection

The spreadsheet implements robust error handling for formula parsing and evaluation. It detects circular references before they cause infinite loops, providing clear feedback to users about the problem and highlighting the affected cells.

Undo & Redo Functionality

All actions in the spreadsheet can be undone and redone, providing users with flexibility and confidence when making changes. The application maintains a history stack of operations allowing for multiple levels of undo/redo.

Multi-cell Selection & Highlighting

Users can select multiple cells using shift-click or by dragging across the grid. The application also highlights cells being referenced by formulas, helping users understand dependencies between cells.

Save & Open Functionality

Spreadsheets can be saved to and opened from XML files, preserving all cell data, formulas, and formatting. The XML structure is designed to be both human-readable and efficient for parsing.

Development Approach

The application was developed using Test-Driven Development (TDD) principles, ensuring high code quality and thorough test coverage. StyleCop was used to maintain consistent code style and readability across the codebase.

The project follows the Model-View-ViewModel (MVVM) architecture pattern, separating the UI logic from the business logic and making the code more maintainable and testable.

Challenges & Solutions

Formula Evaluation Challenge: Creating a robust formula parser that could handle complex expressions while maintaining good performance was challenging. The solution was to implement a custom expression evaluator that converts formulas into abstract syntax trees.

Circular Reference Detection: Detecting circular dependencies between cells required careful implementation. I developed a directed graph approach to track cell dependencies and run cycle detection algorithms before evaluating formulas.

Cross-Platform UI: Ensuring consistent UI behavior across different operating systems was solved by using Avalonia UI, which provides a unified API for Windows, macOS, and Linux platforms.

Outcome

The result is a streamlined yet powerful spreadsheet application with a focus on core functionality, stability, and cross-platform compatibility. The clean architecture and comprehensive test coverage make it easy to maintain and extend with new features.
